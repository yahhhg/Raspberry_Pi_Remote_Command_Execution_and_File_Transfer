# 构建：

### 一、环境要求

上位机（Windows 端）：

- 操作系统：Windows 10 / Windows 11（64 位）
- 开发环境：Qt 6.5+（推荐 LTS 版本，如 6.6.3 LTS）
- 编译器：MinGW 64-bit 或 MSVC 2019/2022 64-bit
- 依赖 Qt 模块：`Core` `Gui` `Widgets` `Network` `Concurrent`

下位机（树莓派端）：

- 硬件：树莓派全系列型号（推荐树莓派 3B+/4B/5）
- 操作系统：Raspberry Pi OS (Debian 11/12)、Ubuntu for ARM 等主流 Linux 发行版
- 软件环境：Python 3.8+
- 依赖 Python 库：均为 Python 标准库（`socket` `subprocess` `os` `threading` `time` `shutil`），无需额外 pip 安装

### 二，核心参数配置

编辑下位机 Python 源码文件，修改上位机的 IP 地址与端口配置，确保与上位机一致：

```
# 核心配置项（请替换为你的实际环境参数）
HOST = "192.168.xxx.xxx"  # 改为 Windows 上位机的局域网 IP 地址
PORT = 8888                # 与上位机默认监听端口保持一致
```

# 项目功能：

此项目为**基于TCP协议的树莓派远程控制器**，分为上位机端（Windows Qt图形化程序）和下位机端（树莓派Python程序），实现Windows主机对树莓派的远程管理、命令交互与文件双向传输：

上位机端：

1. **TCP服务管理**：一键启动/停止TCP服务，默认监听8888端口，支持单客户端接入，实时显示服务状态与客户端连接信息
2. **远程Shell命令交互**：支持输入Linux Shell命令，一键发送至树莓派执行；实时接收并可视化展示命令执行结果、错误信息，支持回车快捷发送，且可以随时停止执行
3. **双向文件传输**：

   - 向下位机发送文件：图形化文件选择对话框，一键发送本地文件到树莓派，带实时传输进度条显示
   - 从下位机拉取文件：输入树莓派端文件路径，即可请求并接收远程文件，自动保存到本地系统下载目录
4. **线程安全架构**：网络通信、文件传输逻辑运行在独立子线程，完全避免UI主线程阻塞（文件选择弹窗、界面操作不影响网络心跳与数据收发）
5. **心跳保活适配**：自动响应下位机心跳包，维持长连接，客户端异常断开实时更新UI状态并释放资源

下位机端（树莓派）：

1. **自动重连TCP客户端**：主动连接上位机服务端，连接失败、异常断开后自动5秒重试重连
2. **远程命令执行引擎**：接收上位机下发的Shell命令，在树莓派本地执行；完整捕获标准输出、错误输出、执行超时，统一格式化后返回结果
3. **双向文件传输适配**：

   - 响应上位机文件拉取请求：读取指定路径的本地文件，分块传输给上位机，同步发送文件元数据与传输结束标记
   - 接收上位机发送的文件：按协议接收文件元数据、二进制流，自动保存到本地`downloads`目录，传输异常时自动清理不完整文件
4. **智能心跳保活**：每隔5秒向上位机发送心跳包检测连接状态，无响应超时自动重连；文件传输期间自动适配保活逻辑，避免弹窗停留、大文件传输时误断连
5. **TCP粘包处理**：基于4字节大端序长度前缀的自定义通信协议，彻底解决TCP粘包问题，保证数据收发完整、解析准确
6. **全链路异常容错**：网络异常、文件操作错误、命令执行超时等全场景异常捕获与兜底处理，不会导致程序崩溃，始终维持可重连状态

# 主要实现：

### TCP 服务器的操作：

当用户按下开启服务器的按钮时 `toggleServer()`将负责创建/销毁服务器对象，其检测服务器对象 m_server 的存活来决定创建和释放，并牵连部分 UI 的更新；创建服务器后，将监听 8888 端口；

但有新连接信号发出后 (QTcpServer::newConnection) ，`onNewConnection()` 将获取第一个连接，并判断客户端处理器是否存活，若存在则弹窗退出（仅允许一个客户端连接），反之则创建线程并实例化客户端处理器，将其移动到线程中，并连接各种信号与槽进行线程间通信，实现数据收发，连接建立断开等操作；最后启动线程并更新 UI；

服务器关闭或客户端断联后，将停止线程运行，并设置超时机制 `m_clientThread->wait(3000)`等待3秒，超时则强制终止线程；最后将指针置空并更新 UI；


### 远程命令行操作：

当用户点击命令行发送按钮或按下 Enter 键后，`sendCommandClicked()` 将获取命令行输入文本并去两端空白字符，将文本包装在信号`sendCommand(cmd)`中发送给客户端处理器；

客户端处理器类接收信号后，执行 `sendCommand(const QString& cmd)` 槽将命令文本封装为 JSON 对象并传入 `sendData(obj)`发送数据包函数，其将计算 JSON数据包 的长度生成长度数据包并先于 JSON 数据包 发送（长度前缀法处理粘包）；

树莓派接收到数据包后，调用`recv_data(sock)`方法，先在缓冲区中接收四个字节的长度前缀数据，再使用 struct 模块将二进制字节流解包为整数（int正好4字节），接着根据这个长度值循环读取（每次读取一个指定大小 4KB）JSON 数据值，最后转 UTF-8 编码后封装进 json 模块对象中返回；而后程序判断 JSON 数据的 type 键值，若 `data["type"] == "cmd"` 则执行`execute_command(data["data"])`传入数据键执行命令，其调用相关接口在树莓派中启动一个shell进程执行；

这个shell进程产生的结果将由主循环调用 `poll_command_output()` 方法返回，其检查当前命令进程存活并实时读取输出，最后在主循环中将这些输出调用`recv_data(sock)`发送给上位机，其也使用长度前缀法，拼接4字节长度数据+JSON数据包发送；

上位机接收数据包时，也是先在缓冲区中接收四个字节的长度前缀数据，再根据这个长度值读取缓冲区数据（不足则等待），最后将接收的数据 QByteArray 转化为 JSON 对象，判断其的 type 键值，`if (type == "result" || type == "error")`则将结果或错误输出到UI上，通过发送信号。


### 发送文件到树莓派：

当用户点击文件发送按钮时，将弹出文件选择框获取选择的文件路径，包装到信号中发送给客户端处理类；

客户端处理器接收到信号后，执行 `sendFile()` 槽，其先发送一个通知包通知树莓派端要接收文件，接着发送 **元数据先导包+文件实际数据+结束包**；树莓派解析通知包后，执行`recv_file()`，创建文件并循环读取文件数据写入，接收后发送消息由上位机接收后再屏幕显示；

在发送期间，上位机每发送一个 4KB 缓冲区的数据，就发送信号更新一次进度条，触发 `updateProgress()` 暂时显示进度条并设置进度。


### 从树莓派接收文件：

用户在输入框中输入树莓派文件的实际路径来获取文件，其将获取去两端空白的文件路径，包装为信号发送给客户端处理器；

客户端处理器接收到信号后，执行 `requestFile()` 槽，其将发送通知包到树莓派端，树莓派发送 **元数据先导包+文件数据+结束包**；上位机按照一般步骤读取文件写入指定目录中。


### 心跳保活机制：

再树莓派与主机建立连接后，将在主循环中每隔一段时间发送心跳保活包给上位机，若上位机在指定时间内未回应，树莓派将跳出主循环，断开连接；其包也是封装为 JSON 数据发送，在上位机中按照正常流程解析；
